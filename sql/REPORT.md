# Executive Summary — Olist E-commerce SQL Analytics

**Format:** SCR (Situation / Complication / Recommendation)  
**Period:** January 2017 – July 2018  
**Analyst:** Sasha  
**Dataset:** Brazilian E-Commerce Public Dataset by Olist

---

## S — Situation

Проведено SQL-аналіз бразильського e-commerce маркетплейсу Olist за період січень 2017 — липень 2018. Аналіз охоплює 96,478 доставлених замовлень та 93,358 унікальних клієнтів. Дані до 2017 року виключені з аналізу — бізнес знаходився на стадії запуску і не відображають повну картину.

Три напрямки аналізу:
- Динаміка revenue та категорійна ефективність
- Профіль клієнтської бази (repeat rate, RFM сегментація)
- Когортний retention

---

## C — Complication / Findings

### 📈 Revenue та MoM динаміка

Загальна динаміка revenue позитивна — зростання з ~$40K (жовтень 2016) до ~$977K (травень 2018). Проте з початку 2018 року бізнес **вперся в плато біля $1M на місяць** — стабільного зростання немає.

Місяці з від'ємним MoM growth:
- Квітень 2017: **-5.2%**
- Червень 2017: **-13.8%**
- Грудень 2017: **-26.5%** *(post-Black Friday correction — очікувано)*
- Лютий 2018: **-10.6%**
- Червень 2018: **-12.4%**

Виняток: листопад 2017 показав **+52.4% MoM** — явний Black Friday ефект. Грудневе падіння після нього є нормальним сезонним патерном, а не сигналом проблеми.

Місяці з від'ємним MoM поза сезонними поясненнями потребують перевірки маркетингових активностей у відповідні періоди.

### 👥 Клієнтська база — Repeat Rate та RFM

**Repeat Purchase Rate: 3%**

З 93,358 унікальних клієнтів лише **2,801 зробили більше однієї покупки**. 97% — one-time buyers.

Це не обов'язково провал retention-стратегії — це структурна характеристика marketplace-моделі. Проте з точки зору unit economics це критична проблема: залучення нового клієнта коштує дорожче ніж повернення існуючого.

**RFM Сегментація:**

| Сегмент | Клієнтів | % від бази |
|---|---|---|
| Loyal | 45,099 | 48.3% |
| Others | 23,368 | 25.0% |
| Lost | 11,705 | 12.5% |
| Recent | 11,607 | 12.4% |
| Champion | 1,579 | 1.7% |

- **Loyal (48%)** — найбільший сегмент. Купують регулярно але не досягли рівня Champions
- **Champions (1.7%)** — VIP клієнти з найвищими R, F, M scores. Мала частка при великій базі
- **Lost (12.5%)** — 11,705 клієнтів які давно не повертались
- **Others (25%)** — неоднорідна група, потребує додаткової сегментації

---

### 🔄 Когортний Retention

Retention підтверджує висновок про one-time buyer модель. Більшість когорт показують **<1% retention** вже з першого місяця після покупки. Повторні покупки в межах одного клієнта трапляються але носять нерегулярний характер (gap між місяцями 6, 9, 11 замість послідовних 1, 2, 3).

Патерн стабільний по всіх когортах — це системна характеристика, не аномалія окремих місяців.

## R — Recommendation

### 1. Програма конвертації Loyal → Champion
45K Loyal клієнтів — готова аудиторія для апселу. Якщо конвертувати **5% Loyal в Champions**, це +2,250 VIP клієнтів з AOV ~$199+. Запустити персоналізовані офери для Loyal сегменту на основі їх категорійної історії. **Відповідальний:** Marketing team. **Термін:** Q1 наступного кварталу.

### 2. Реактивація Lost сегменту
11,705 Lost клієнтів — це існуюча база з нульовими витратами на залучення. Запустити email/push кампанію з персональними знижками або нагадуванням про категорії які вони купували. Навіть **10% реактивація дасть +1,170 повернень**. **Відповідальний:** CRM team. **Термін:** найближчий місяць.

### 3. Масштабування relogios_presentes та cool_stuff
Обидві категорії мають високий AOV ($199 і $164) але обмежений обсяг замовлень. Перевірити чи є supply-side обмеження (мало продавців) або маркетингова недоінвестованість. Збільшення видимості цих категорій може підняти загальний AOV без збільшення кількості замовлень.

### 4. Аудит маркетингових активностей у місяці з від'ємним MoM
Квітень 2017, червень 2017, лютий 2018, червень 2018 показали падіння не пов'язане з сезонністю. Запросити у marketing team дані по витратах і каналах за ці місяці для виявлення кореляції з падінням revenue.

### 5. Стратегія repeat purchase
97% one-time buyers — фундаментальна проблема unit economics. Розглянути механіки підвищення repeat rate: підписки, loyalty points, категорійні бандли, post-purchase email sequences з релевантними рекомендаціями.

## ⚠️ Limitations

- Дані серпня 2018 неповні — останній місяць виключений з MoM висновків
- Дані 2016 року виключені через неповноту (стадія запуску бізнесу)
- Olist — marketplace, не прямий ритейлер. Низький repeat rate частково пояснюється структурою платформи: клієнт може повернутись через іншого продавця і не бути ідентифікований як repeat buyer
- RFM сегментація на основі квартильних розрізів (NTILE), не валідованих бізнес-порогів — сегменти є відправною точкою для гіпотез, не фінальною класифікацією
- Відсутні дані по маркетингових витратах і каналах залучення — неможливо розрахувати CAC і ROAS
